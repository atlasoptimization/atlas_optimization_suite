# Asset Footprint Policy

This document describes the intended asset layers for the Data Science Deck app. It is a policy for review and cleanup planning, not a cleanup action by itself.

## Asset Layers

### Source full assets

The full-resolution source assets live outside the app repository, for example:

- `/home/jemil/Desktop/Card_game_full/Scribus_setup/Scribus_files/output`
- `/home/jemil/Desktop/Card_game_full/Scribus_setup/Scribus_files/images`

These are sync inputs. The browser should never load these paths directly, and runtime manifests must never contain private absolute filesystem paths.

### Compressed source assets

Compressed PDFs and images are also outside the app repository, for example:

- `/home/jemil/Desktop/Card_game_full/Scribus_setup/Scribus_files/output_compressed`
- `/home/jemil/Desktop/Card_game_full/Scribus_setup/Scribus_files/images_compressed`

These are preferred sync inputs for public deployment. They should be used to produce the browser runtime bundle, not loaded directly by the browser.

### Runtime public assets

`apps/web/public/deck` is the browser runtime bundle. Vite copies this folder into the build output, and GitHub Pages serves these files publicly.

For GitHub Pages, this folder must contain only browser-safe assets and paths. It may include:

- compressed PDFs for Open PDF
- card-face previews for desk display
- card art images for inspector/browser use
- backgrounds, handbook, icons, and approved custom-domain assets if intended for the public demo
- `manifest.json` with relative public paths such as `deck/...`

### Build output

`apps/web/dist` is build output. It is generated by Vite and should be reproducible from source code plus `apps/web/public/deck`.

GitHub Actions deploys `apps/web/dist`. It should not run private sync steps and should not need `.env.local`.

## Deployment Intent

For local full development, sync may use full or compressed assets depending on local configuration.

For GitHub Pages, prefer compressed assets:

- compressed PDFs for Open PDF
- one card-face preview set suitable for desk display
- compressed WebP art images for inspector and browser views

Avoid committing redundant forms unless there is a clear runtime need.

## Avoid Redundant Runtime Assets

Avoid keeping multiple public runtime forms of the same content when one form is sufficient:

- original PDFs plus compressed PDFs
- PNG previews plus WebP previews
- stale thumbnail/reading preview variants that the manifest no longer references
- stale custom-domain assets in the public bundle
- old files left behind after changing preview format or image extension mapping

If both thumbnail and reading previews are retained, document why both are needed. The expected reason is: thumbnails for small UI lists and reading previews for desk cards.

## Cleanup Rules

No cleanup should happen before running:

```bash
pnpm audit:asset-inventory
pnpm audit:asset-cleanup-plan
```

Cleanup should be done in a separate branch after reviewing `reports/asset-cleanup-plan.md`.

Do not delete source full assets or compressed source assets from outside the app as part of app cleanup.

Do not rewrite Git history as part of normal asset cleanup. Git history cleanup is separate, dangerous, and should happen only after a backup and a deliberate plan.

## Public Safety

The release audits should continue to reject private paths such as:

- `/home/`
- `/Desktop/`
- `file://`
- `Card_game_full`
- `Scribus_setup`
- `/mnt/data/`

Everything in `apps/web/dist` is public and downloadable after deployment. The path audits prevent private filesystem paths from leaking, but they do not decide whether card content is appropriate for public release.
